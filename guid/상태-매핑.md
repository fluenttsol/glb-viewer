
# 눈 (Eye Expressions)

모든 눈 표정은 coverage 함수 기반. 표정 간 전환은 파라미터 보간으로 부드럽게 처리.

| # | preset | mood | 설명 | 모션 |
|---|--------|------|------|------|
| 1 | Neutral | `neutral` | 기본 뜬 상태. 캡슐형 아웃라인 + 동공 | 없음 (정적) |
| 2 | Blink | `blink` | 감은 상태. 아래로 볼록한 곡선 (눈웃음) | 자동 깜빡임 타이머로 open 0→1 보간 |
| 3 | Twinkle | `twinkle` | neutral + 동공 위 별 하이라이트 | sparkle 파라미터 |
| 4 | Loading | `loading` | 동그란 아웃라인 (로딩 인디케이터) | 없음 (정적) |
| 5 | Moving | `moving` | neutral 모양 + 눈동자 좌우 구르기 | sin/cos 기반 shiftX/Y → atlas box 이동 |
| 6 | Die | `die` | X자 모양 | scanlineGlitch (LCD 지지직 효과) |
| 7 | Wink | `wink` | 한쪽 감고 한쪽 뜸. 감은 쪽 아래 볼록 곡선 | winkSide로 좌/우 지정 |
| 8 | Sleepy | `sleepy` | 반쯤 감긴 눈. 위로 볼록 곡선 (무거운 눈꺼풀) | sin+pow 기반 꾸벅꾸벅 조는 모션 |

- 모든 전환은 `blendEyePoses`로 파라미터 보간 (하드 분기 없음)
- `closeFactor` 블렌딩으로 open→closed 연속 전환
- `blinkCurveDir`: 양수 = 눈웃음, 음수 = 무거운 눈꺼풀
- 시선 이동: atlas box 위치로만 표현 (coverage 내부 shift 없음 → 잘림 방지)


# 입 (Mouth Expressions)

모든 입 표정은 coverage 함수 기반. 표정 간 전환은 coverage grid 블렌딩.

| # | preset | mood | 설명 |
|---|--------|------|------|
| 1 | Rest | `rest` | 일자 입. 가로로 긴 2줄 직선 |
| 2 | Smile | `smile` | 아래로 볼록한 곡선 (미소) |
| 3 | O | `o` | 동그란 O 모양 (놀람/감탄) |
| 4 | Ah | `ah` | 아래로 둥근 D자 형태. 채워진 모양 (크게 벌린 입) |
| 5 | Sad | `sad` | 위로 볼록한 곡선 (슬픔) |
| 6 | Woo | `woo` | 작은 동그란 입 (우~ 발음) |

- 전환: `blendMouthCoverage`로 from/to coverage grid 보간 후 양자화
- 입은 눈 시선(sharedGaze)의 50% 비율로 따라감


# 전체 상태 매핑

시스템 상태별 눈/입/바디 조합. 우선 매핑 대상: idle, thinking, done, searching, error, offline, talking

| 상태 | 아스키 눈 | 아스키 입 | 바디 모션 | 근거 |
|---|---|---|---|---|
| `idle` | Neutral + 자동 Blink | Rest | idle / breathing | 기본 대기 상태. 주기적 깜빡임으로 생동감 |
| `thinking` | Moving | Rest | 추가 필요 | 눈동자가 이리저리 굴러가며 생각하는 느낌 |
| `done` | Neutral + Blink → Wink | Smile | jump | 완료 만족감. 윙크로 "다 했어!" 느낌 |
| `searching` | Loading | O | 추가 필요 | 로딩 아웃라인 눈 = 검색/탐색 중 |
| `error` | Die | Sad | waving | X눈 + 슬픈 입 = 에러/실패 상태 |
| `offline` | Sleepy | Rest | standing react death backward | 꾸벅꾸벅 조는 눈 = 오프라인/비활성 |
| `talking` | Neutral + 자동 Blink | Ah ↔ Rest 반복 | talking | 말하는 동안 입이 벌어졌다 닫혔다. 자연스러운 깜빡임 유지 |

## 추가 고려 상태 (후순위)

| 상태 | 아스키 눈 | 아스키 입 | 바디 모션 | 근거 |
|---|---|---|---|---|
| `reading` | Neutral (시선 약간 아래) | Rest | 추가 필요 | 읽는 중 = 집중한 눈. shiftY로 아래 응시 |
| `writing` | Neutral (시선 아래) | Rest | 추가 필요 | reading과 유사하지만 더 집중된 느낌 |
| `terminal` | Neutral | Rest | 추가 필요 | 코드 실행 대기 |
| `browsing` | Moving (느린 속도) | Rest | 추가 필요 | 천천히 둘러보는 눈동자 |
| `delegating` | Wink | Smile | 추가 필요 | "맡겨둬!" 느낌의 윙크 |
| `waiting_permission` | Neutral + 자동 Blink | O | waving | 허락 기다리며 약간 놀란 입 |
| `rate_limited` | Sleepy | Sad | surprised | 지쳐서 쉬는 느낌 |
| `pending` (미연결) | Sleepy | Rest | floating | 아직 연결 안 됨 = 떠다니며 졸기 |

## 구현 참고

- 상태 전환 시 눈/입 모두 블렌딩으로 부드럽게 전환 (이미 구현됨)
- `talking` 상태의 입 모션은 Ah ↔ Rest를 주기적으로 전환하는 타이머 필요
- 바디 모션은 GLB 애니메이션 클립명과 매핑 필요 (런타임 로드)
- scanlineGlitch는 error 외에도 glitch 연출이 필요한 상태에서 재사용 가능
